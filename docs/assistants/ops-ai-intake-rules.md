# 运营负责人 AI 信息入口规则

这份规则用于定义运营负责人从钉钉、企微、飞书或 Codex 发给 AI 的信息如何被识别、存储、提醒和进入日报。

## 入口原则

- 先保证信息能被可靠收集，再追求多平台自动同步。
- 先做归档、摘要、提醒和草稿，不直接替人做高风险动作。
- 每条信息都要能追溯来源、时间、原文和处理状态。
- 涉及客户承诺、合规、财务、审批、生产数据、对外口径的事项必须人工确认。

## 信息类型

| 类型 | 触发语或特征 | 存放位置 | 默认处理 |
| --- | --- | --- | --- |
| 待办 | “提醒我”“跟进”“安排”“下周处理” | 待办池 | 提取时间、对象、动作，等待确认后创建提醒 |
| 灵感 | “可以试试”“想到一个点子”“增长机会” | 灵感池 | 记录背景，建议是否转为实验卡片 |
| 资料 | 链接、文件、截图、长文本 | 资料池 | 摘要重点，标注用途和相关主题 |
| 决策 | “决定”“先按”“确认走” | 决策池 | 记录背景、影响范围和后续动作 |
| 风险 | “注意”“有风险”“别自动发”“涉及合规” | 风险池 | 标为人工确认，不自动执行 |
| 客户信号 | 客户反馈、投诉、需求、续费、流失风险 | 客户信号池 | 归类为问题、需求、风险或机会 |
| 实验想法 | 新渠道、新活动、新产品尝试、新转化路径 | 实验池 | 生成增长实验卡片草稿 |

## 推荐发送格式

为了减少歧义，可以使用短标签。标签不是强制的，但会提升识别准确率。

```text
[待办] 明天下午提醒我跟进 A 客户续费方案
[灵感] 可以把客户服务里的高频问题做成一个公开 FAQ
[风险] 这个回复可能涉及效果承诺，先不要自动发送
[实验] 试一下把新用户引导页改成行业场景入口
[资料] 这份竞品资料帮我归档到市场需求
```

## 处理状态

每条信息建议有 6 种状态：

| 状态 | 含义 |
| --- | --- |
| 已收集 | AI 已接收但尚未处理 |
| 已归类 | 已判断类型和主题 |
| 待确认 | 需要负责人确认后才能继续 |
| 已转行动 | 已生成待办、实验卡片、草稿或文档 |
| 已进入日报 | 已在当日 AI 日报中体现 |
| 已关闭 | 已完成、归档或确认无需处理 |

## 人工确认规则

以下事项默认进入 `待确认`：

- 对外发送客户消息。
- 承诺交付结果、收益、时间或补偿。
- 解释合规、合同、法务、财务问题。
- 创建、取消或调整正式审批。
- 修改生产数据或客户关键资料。
- 群发通知、公告或对外口径。
- 将内部信息转发到外部平台。

## 进入日报的规则

不是所有信息都进入日报。优先进入日报的是：

- 影响收入、用户增长、客户服务、合规或增长实验的信息。
- 当天形成的关键判断、决策或风险。
- 需要明天跟进的事项。
- 可以沉淀为模板、脚本、Skill 或机器人能力的事项。

低价值、重复、已完成且无复盘价值的信息只归档，不进入日报正文。

## 第一阶段平台策略

### 钉钉

适合承接：

- 工作群消息摘要。
- 机器人提醒。
- 任务和日报入口。
- 团队推广时的通知触达。

第一阶段建议：先使用指定群或机器人接收转发信息，避免一开始接入所有群。

### 飞书

适合承接：

- 文档、Base、日报、知识库。
- 日程和任务。
- 结构化复盘和长期沉淀。

第一阶段建议：将 AI 日报、实验卡片、调研清单和周复盘沉淀在飞书文档或 Base。

### 企微

适合承接：

- 客户沟通线索。
- 社群反馈。
- 销售或客户成功跟进信号。

第一阶段建议：先通过人工转发或摘要输入，不直接自动读取和回复客户消息。

### Codex

适合承接：

- 深度规划。
- 文档和代码沉淀。
- 自动化脚本、Skill 和流程实现。
- 日报模板与助手设计迭代。

## 日报前检查清单

生成每日 AI 日报前，检查：

1. 今天是否有收入或增长相关事项。
2. 今天是否有客户服务或客户风险信号。
3. 今天是否有合规、承诺或数据权限风险。
4. 今天是否有 0-1 实验想法或市场需求信号。
5. 今天是否有团队效率问题。
6. 今天是否有值得沉淀成小助手的重复工作。
7. 明天建议是否少而清楚，最好不超过 3 条。
