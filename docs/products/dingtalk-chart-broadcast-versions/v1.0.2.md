# 钉钉图表播报产品文档

当前产品版本：`v1.0.2`

历史版本快照目录：[dingtalk-chart-broadcast-versions](dingtalk-chart-broadcast-versions/)

文档维护规则：当前文件只维护最新版本说明；每次规划或发布新版本前，必须先把上一版本完整复制到版本快照目录，例如 `dingtalk-chart-broadcast-versions/v1.0.1.md`。历史版本只新增，不覆盖。

## 1. 产品定位

钉钉图表播报用于把监控预警、经营数据、运营报表等结构化信息，通过钉钉机器人稳定推送到指定单聊或群聊。当前落地版本以「退费率播报」为样板，支持 Markdown 卡片和图片表格两种展示方式。

核心目标：

- 让长报表在钉钉客户端完整可读，避免普通文本消息只显示标题、Markdown 不渲染、内容被拆段等问题。
- 面向多人阅读时优先使用图片表格，保证列结构、配色、行高、重点标记一致。
- 保留 Markdown 与图片两种模式，便于图片生成或上传异常时快速回退。

## 2. 产品可选项：Markdown 文本模式与图表模式

钉钉播报能力长期保留两种展示模式，产品设计、需求评审和实现时都需要显式选择。

### 2.1 Markdown 文本模式

适用场景：

- 内容较短，以结论、摘要、异常说明为主。
- 接收人需要直接复制文本。
- 指标行数少，不需要横向对比大量企业或渠道。
- 图片生成、上传或图片清晰度暂不稳定，需要快速回退。

展示规范：

- 仍然使用钉钉互动卡片 `StandardCard`，不要回退到普通文本消息。
- 正文组件必须使用 `contents[].type = "markdown"`。
- 标题、时间、分段标题使用 DingTalk `sizeToken` 字号规范。
- 需要可见空行时使用独立 `&nbsp;` 行，不依赖连续空行或多个 `<br/>`。
- 内容保持一整段卡片正文，不拆成多条机器人消息。

### 2.2 图表模式

适用场景：

- 行列较多，需要多人快速扫读。
- 需要稳定展示表头、列宽、斑马纹、条件色和排序。
- 内容不适合在手机或桌面端用长 Markdown 阅读。
- 需要把复杂表格作为图片沉淀、转发或截图。

展示规范：

- 卡片 Markdown 正文只保留标题、时间和图片。
- 复杂表格用 SVG 渲染为 PNG，再通过互动卡片发送。
- 图片内部包含标题、时间、口径说明和表格。
- 表格必须使用 `clipPath` 防止文本遮挡。
- 需要保留 Markdown 模式作为配置回退，不删除 Markdown 生成逻辑。

### 2.3 默认选择规则

- 默认优先图表模式：面向运营、管理、多人阅读的长报表。
- 默认优先 Markdown 文本模式：短预警、单客户说明、异常摘要、需要复制粘贴的结论。
- 如果用户明确指定“Markdown 文本”或“图表模式”，按用户指定执行。
- 如果图表模式失败，不静默降级；记录错误并提示，再由配置切换到 Markdown 模式。

## 3. 当前样板：退费率播报

### 3.1 推送对象

- 当前机器人：小灰龙-运营助手。
- 当前默认发送方式：钉钉互动卡片 `StandardCard`。
- 当前默认展示模式：图片表格。
- 当前发送目标支持单聊、群聊或双发；群聊使用钉钉群 `openConversationId`，不是群名或普通群号。

### 3.2 数据口径

数据统计范围为今日 `00:00` 至当前整点。对比数据取昨日 `00:00` 至昨日同一整点。

`00:00:00` 至 `00:59:59` 是特殊整点窗口：统计昨日全天，对比前日全天，避免查询今日 `00:00-00:00` 空窗口。

图片表格第三行展示全日参考值：前日全天与昨日全天，只展示数值，不计算环比。DataFinder 全日参考查询必须使用单日 `day` range，例如查询 `2026-06-19` 全天时使用 `range=[2026-06-19 00:00, 2026-06-19 00:00]`，不能使用 `range=[2026-06-19 00:00, 2026-06-20 00:00]`，否则可能被算成两天合计。

非 00 点图片内口径说明为：

```text
口径说明：数据统计范围为今日 00:00 至当前整点；括号内依次为昨日同整点值、环比变化，退费率另展示百分点差异（pp）。
```

00 点特殊窗口图片内口径说明为：

```text
口径说明：数据统计范围为昨日全天；括号内依次为前日全天值、环比变化，退费率另展示百分点差异（pp）。
```

### 3.3 字段定义

图片表格列顺序固定为：

```text
企业名称 | 退费率 | 退费后金额 | 支付金额 | 支付数 | c0收入 | 退费金额 | 退费数
```

字段口径：

- `企业名称`：DataFinder 聚合后的企业简称。
- `退费率`：`退费金额 / 支付金额`。支付金额为 0 且退费金额大于 0 时展示 `无法计算`。
- `退费后金额`：`支付金额 - 退费金额`。
- `支付金额`：`order_suc_back.amount` 的 DataFinder 合计值，使用 `event_indicator="measure"` 与 `measure_info.measure_type="sum"`。
- `支付数`：`order_suc_back` 事件数，使用 `event_indicator="events"`。
- `c0收入`：`order_suc_back.amount` 的 DataFinder 合计值，并过滤 `cycle = 0`。
- `退费金额`：`refund_back.amount` 的 DataFinder 合计值，使用 `event_indicator="measure"` 与 `measure_info.measure_type="sum"`。
- `退费数`：`refund_back` 事件数，使用 `event_indicator="events"`。

过滤规则：

- 过滤 `支付金额 = 0 且退费金额 = 0` 的企业。
- 保留 `支付金额 = 0 且退费金额 > 0` 的企业，并在退费率列展示 `无法计算`。

排序规则：

- `总计` 行固定第一行。
- 企业行按当前 `退费后金额` 降序排列。
- 退费后金额相同时，按企业名中文排序兜底。

### 3.4 单元格展示

金额和数量类指标：

```text
当前值
（昨日同整点值，环比变化）
前日 全日值｜昨日 全日值
```

示例：

```text
992,870.80
（833,936.90，+19.06%）
前日 833,936.90｜昨日 993,328.00
```

退费率指标：

```text
当前退费率
（昨日同整点退费率，环比变化 / 百分点差异）
前日 全日退费率｜昨日 全日退费率
```

示例：

```text
17.03%
（21.36%，-20.27% / -4.33pp）
前日 19.06%｜昨日 16.49%
```

如果昨日同整点值为 0 或缺失，只展示当前值，不展示括号。第三行前日或昨日全日参考值缺失时，只隐藏缺失项，不影响同单元格其他行。

## 4. 卡片与图片样式规范

### 4.1 卡片正文

图片模式下，卡片 Markdown 正文只保留：

```text
退费率播报
时间: YYYY-MM-DD HH:mm:ss
图片
```

不在图片上方重复展示「总览」「异常总结」等长文字，避免占用首屏和造成阅读噪音。

### 4.2 图片结构

图片内部从上到下依次为：

1. 报表标题。
2. 报表时间。
3. 口径说明。
4. 表格。

### 4.3 配色

默认配色：

- 页面背景：`#ffffff`。
- 表头背景：`#B5C6EA`。
- 表头文字：`#111827`。
- 总计行背景：`#eef6ff`。
- 普通行背景：`#ffffff` / `#f7f7f8` 斑马纹。
- 关注上升：浅红背景。
- 关注下降：浅绿背景。
- 支付为 0 但有退费：浅黄背景。

### 4.4 防遮挡规则

- 图片由 SVG 渲染为 PNG。
- 每个表格单元格必须设置 `clipPath`，防止长文本串到相邻列。
- 指标值、昨日同整点对比值、前日/昨日全日参考值拆成三行展示。
- 新增列后需要同步调整列宽，优先保证金额类列不遮挡。

## 5. 发送链路

### 5.1 推荐链路

使用钉钉互动卡片：

- OpenAPI：`/v1.0/im/v1.0/robot/interactiveCards/send`
- 模板：`StandardCard`
- 正文组件：`contents[].type = "markdown"`
- 机器人：使用 `DINGTALK_BOT_ID` 对应 robot code。
- 单聊目标：请求体使用 `singleChatReceiver`。
- 群聊目标：请求体使用 `openConversationId`。

图片模式下：

1. 查询 DataFinder 数据。
2. 生成报表数据结构。
3. 用 SVG 生成 PNG 表格。
4. 上传 PNG 到钉钉媒体接口。
5. 在 `StandardCard` Markdown 中插入图片。
6. 发送到指定单聊用户或群会话。

### 5.2 不推荐链路

不推荐使用普通文本消息发送长报表：

- `send-by-bot --text` 在单聊里可能只显示标题或首行。
- 普通文本不会稳定渲染 Markdown。
- 长文本拆段后阅读体验差，也容易造成上下文丢失。

## 6. 配置项

关键配置：

```text
DINGTALK_CLIENT_ID
DINGTALK_CLIENT_SECRET
DINGTALK_BOT_ID
REFUND_REPORT_DELIVERY_TARGET=single|group|both
REFUND_REPORT_USER_IDS
REFUND_REPORT_GROUP_CONVERSATION_ID
REFUND_REPORT_CARD_TEMPLATE_ID=StandardCard
REFUND_REPORT_RENDER_MODE=markdown|image
REFUND_REPORT_THRESHOLD_PERCENT=10
```

模式选择：

- `REFUND_REPORT_RENDER_MODE=image`：发送图片表格卡片，适合长报表和多人阅读。
- `REFUND_REPORT_RENDER_MODE=markdown`：发送 Markdown 卡片，适合快速回退或短报表。

发送目标：

- `REFUND_REPORT_DELIVERY_TARGET=single`：发送到 `REFUND_REPORT_USER_IDS` 配置的单聊用户，默认兼容旧行为。
- `REFUND_REPORT_DELIVERY_TARGET=group`：发送到 `REFUND_REPORT_GROUP_CONVERSATION_ID`；未配置时回退 `DINGTALK_DEFAULT_GROUP_CONVERSATION_ID`。
- `REFUND_REPORT_DELIVERY_TARGET=both`：同时发送群聊和单聊，要求用户 ID 与群 ID 都存在。
- 查询群 `openConversationId` 推荐使用 `dws chat search --keyword "群名关键词" --format json`，从候选群里人工确认目标群。

## 7. 青龙任务部署

云服务器上推荐使用青龙任务调度单次脚本，不启动本地常驻 Stream 服务。

### 7.1 执行入口

本地和云上都可用的单次执行命令：

```bash
npm run refund-report:once
```

青龙任务推荐执行包装脚本：

```bash
bash /ql/data/scripts/project000/scripts/ql_refund_report.sh
```

该脚本会：

- 进入项目目录。
- 执行 `git pull --ff-only` 拉取最新代码。
- 在依赖缺失或 `package-lock.json` 变化时执行 `npm ci`。
- 检查 Python 依赖 `rangersdk`，缺失时安装。
- 执行单次退费率播报脚本。

### 7.2 青龙配置

推荐配置：

```text
任务名称：退费率播报-钉钉卡片
命令：bash /ql/data/scripts/project000/scripts/ql_refund_report.sh
定时：0 * * * *
```

青龙环境变量必填：

```text
DINGTALK_CLIENT_ID
DINGTALK_CLIENT_SECRET
DINGTALK_BOT_ID
REFUND_REPORT_USER_IDS
REFUND_REPORT_GROUP_CONVERSATION_ID
DATAFINDER_APP_ID
DATAFINDER_ACCESS_KEY
DATAFINDER_SECRET_KEY
```

青龙环境变量推荐：

```text
REFUND_REPORT_RENDER_MODE=image
REFUND_REPORT_DELIVERY_TARGET=group
REFUND_REPORT_CARD_TEMPLATE_ID=StandardCard
REFUND_REPORT_TIMEZONE=Asia/Shanghai
REFUND_REPORT_THRESHOLD_PERCENT=10
DINGTALK_API_BASE_URL=https://api.dingtalk.com
```

### 7.3 本地服务与青龙任务边界

- `src/index.ts` 继续用于本地机器人 Stream 对话、群聊总结和本地小时调度。
- 青龙任务只调用 `scripts/send_refund_report_once.mjs` 做一次播报。
- 青龙任务失败时由青龙记录失败日志，不在脚本内静默吞错。

## 8. 异常与回退

图片模式失败时，不静默降级成 Markdown，避免误以为图片版发送成功。

推荐处理：

1. 记录图片生成、上传或卡片发送错误。
2. 发送失败卡片或告警。
3. 如需临时恢复播报，改为 `REFUND_REPORT_RENDER_MODE=markdown` 后重启服务。

## 9. 验收标准

功能验收：

- 卡片能在钉钉客户端完整显示标题、时间和图片。
- 群聊模式下，卡片请求体使用 `openConversationId`，且不包含 `singleChatReceiver`。
- `both` 模式下，群聊和单聊都能收到同一内容，且每条卡片的 `cardBizId` 唯一。
- 图片表格包含 `退费后金额` 列，位置在 `退费率` 后、`支付金额` 前。
- 图片表格包含 `c0收入` 列，位置在 `支付数` 后、`退费金额` 前。
- `退费后金额 = 支付金额 - 退费金额`。
- 企业行按当前 `退费后金额` 降序排列，总计行固定第一行。
- 第三行全日参考值为单日全量，不得出现两日合计；例如 6.19/6.20 宜信退费后金额应为 `833,936.90 / 993,328.00`，奇富数科应为 `680,919.70 / 630,702.30`。
- 金额类字段与 DataFinder UI 的 `合计值` 对齐，使用 `measure sum(amount)`，不得用 `amount * events` 复算。
- 指标括号内不出现 `昨日同期：`、`环比：` 等冗余文字。
- 口径说明位于图片标题/时间下方、表头上方。
- 表头颜色为 `#B5C6EA`。
- 单元格文本不遮挡、不串列。

工程验收：

- 图片模式和 Markdown 模式可通过配置切换。
- 单测覆盖字段顺序、退费后金额计算、`c0收入`、全日参考单日查询、排序、PNG 生成和卡片发送。
- 服务重启后日志显示 `renderMode: image` 或指定模式。

青龙验收：

- 手动运行青龙任务一次，日志出现 `refund_report.once.completed`。
- 钉钉客户端收到一张完整卡片。
- 等待下一个整点，确认任务自动触发成功。

## 10. 后续扩展

后续新增钉钉图表播报时，优先复用本规范：

- 互动卡片承载消息。
- SVG 转 PNG 承载复杂表格。
- 卡片正文只放标题、时间和图片。
- 图片内部放口径说明。
- 表头使用 `#B5C6EA`。
- 长文本使用 `clipPath` 防遮挡。
- 保留 Markdown 回退开关。
