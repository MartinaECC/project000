# 钉钉应用版本记录

本文档记录钉钉应用/播报产品能力版本。产品版本独立于代码包版本，`package.json` 的版本号按工程包发布节奏单独管理。

## 版本规则

- `v1.0.x`：向后兼容增强，例如播报格式、字段、口径、排序、异常规则、文档和 skill 的增强。
- `v1.x.0`：功能版本，例如新增一类完整播报、接入新数据源、新机器人场景、群聊/单聊模式扩展。
- `v2.0.0`：破坏性升级，例如重做卡片模板、替换发送链路、废弃旧配置或改动核心数据口径。

## v1.0.0 - 2026-06-21

### 新增能力

- 建立钉钉互动卡片发送链路，使用 `StandardCard` 发送机器人单聊报表。
- 支持 Markdown 卡片和图片表格两种播报模式，并通过配置切换。
- 建立退费率播报样板，包括标题、时间、口径说明、表格、异常关注和真实卡片验证流程。
- 图片表格支持固定列、表头配色、斑马纹、总计行、条件色、`clipPath` 防遮挡。

### 数据口径

- 退费率播报默认按当日 `00:00` 至当前整点统计，对比昨日同整点窗口。
- `00:00:00` 至 `00:59:59` 是特殊整点窗口：统计昨日全天，对比前日全天。
- 分时对比是通用规则：对比周期必须取对应的同一分时窗口，例如今日 `00:00-09:00` 对比昨日 `00:00-09:00`。
- DataFinder 非全天窗口使用 `granularity: "hour"` 并汇总已完成小时桶；全天窗口使用 `granularity: "day"`。

### 发送目标和触发方式

- 当前目标为小灰龙-运营助手机器人单聊。
- 本地常驻服务每小时整点自动推送。
- 保留单次脚本入口 `npm run refund-report:once`，用于测试卡片和后续调度迁移。

### 配置项变化

- `REFUND_REPORT_RENDER_MODE=markdown|image` 控制 Markdown 或图片表格模式。
- `REFUND_REPORT_CARD_TEMPLATE_ID=StandardCard` 使用标准互动卡片模板。
- `REFUND_REPORT_TIMEZONE=Asia/Shanghai` 控制自然日和整点窗口。
- `REFUND_REPORT_THRESHOLD_PERCENT=10` 控制退费率环比关注阈值。
