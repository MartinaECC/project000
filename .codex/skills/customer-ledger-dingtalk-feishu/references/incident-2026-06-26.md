# 2026-06-26 MVP 联调复盘

## 现象

- 用户在钉钉群 `@小灰龙-运营助手 京东金融 今天15:30 ...`，飞书未新增。
- 初期日志没有任何 `stream.robot.message.received`。
- 后续修正机器人配置后，钉钉能收到回复，但飞书同步失败：`spawn EINVAL`。

## 根因 1：钉钉机器人配置不一致

`start_intake` 使用 `.env.intake.local`，该文件仍配置旧 AppKey/robot code `ding3cv0e5gguron10zy`。

钉钉侧搜索到的真实机器人是：

```json
{
  "robotCode": "dingxuuuvqzncyx1o6wp",
  "robotName": "小灰龙-运营助手"
}
```

修正：从 `.env` 复制 `DINGTALK_CLIENT_ID`、`DINGTALK_CLIENT_SECRET`、`DINGTALK_BOT_ID` 到 `.env.intake.local`，重启 `scripts/start_intake.mjs`。

## 根因 2：Windows 下直接 spawn lark-cli.cmd 失败

Node 24 + Windows 下直接：

```js
spawn('lark-cli.cmd', ['--version'], { shell: false })
```

会抛出：

```text
Error: spawn EINVAL
```

修正：`src/customer-ledger.ts` 的 `runCli` 在 Windows 下对 `.cmd/.bat` 走 `cmd.exe /d /c` 包装。

## 图文登记扩展验证

### 现象

- 用户希望钉钉同条图文消息中的图片也进入飞书客户台账。
- 第一版只识别直接图片 URL；真实钉钉富文本中图片以 `downloadCode` / `pictureDownloadCode` 形式出现，且图片节点中的 token 被错误拼进了正文。
- 第二版能写入图片，但图片位于最右侧 `文档` 列，不符合“紧跟文字之后”的阅读习惯。

### 修正

- `src/stream-adapter.ts`：识别 `richText` 图片节点，提取 `downloadCode` / `pictureDownloadCode`，并避免把图片 token 当成正文。
- `src/dingtalk-media.ts`：调用 `POST https://api.dingtalk.com/v1.0/robot/messageFiles/download`，用钉钉 `downloadCode` 换取临时 `downloadUrl`。
- `src/customer-ledger.ts`：将图片 `<img href="..."/>` 写入 `内容` 单元格，紧跟文字之后，不再写入最后一列。
- `src/bot-service.ts`：图片解析失败时先写 JSONL pending，再标记 failed 并回复用户，不静默丢记录。

### 验证结果

- `#CL-20260626-010`：图文登记成功，图片进入飞书台账；该条为旧布局，图片位于最右侧 `文档` 列。
- `#CL-20260626-011`：图文登记成功，图片位于 `内容` 单元格并紧跟文字之后。
- 目标文档：`005 宜享花｜项目台账`。
- doc token：`WhEOdKEDdotX7VxPRckcFiQsnce`。

### 后续提醒

- 图片必须和客户/日期/动作文字在同一条钉钉消息里。
- 纯图片消息暂不登记，因为无法可靠判断客户和动作。
- 钉钉媒体下载 URL 是临时链接；如果未来发现飞书图片过期，应把图片转存到飞书 Drive 后再写入 Docx。
- 早期调试产生的测试残留行不会自动清理，需要人工删除或后续实现按 block id 清理脚本。

## 团队成员记台账格式修正

### 现象

- `增长运营组` 群成员发送 `@小灰龙-运营助手 记台账： 京东金融 26日 订单可获取到手机号吗`。
- 服务已放开群白名单，消息进入 Stream，但机器人回复了通用 LLM 文案，没有写入飞书台账。

### 根因

- 日志显示该消息 `intentType=unknown`，不是权限问题。
- 解析器只支持 `客户 时间 动作` 和少量 `登记台账：` 前缀，未支持团队成员自然使用的 `记台账：` 前缀。
- 日期解析也未支持 `26日` 这种日内简写。

### 修正

- `src/customer-ledger.ts`：新增 `记台账`、`台账`、`记录台账`、`记账` 等命令前缀剥离。
- `src/customer-ledger.ts`：新增 `d日`、`M月d日` 中文日期格式解析。
- `test/customer-ledger.test.ts`：增加群成员原句回归用例。
