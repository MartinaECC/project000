# 钉钉应用版本记录

本文档记录钉钉应用/播报产品能力版本。产品版本独立于代码包版本，`package.json` 的版本号按工程包发布节奏单独管理。

## 版本规则

- `v1.0.x`：向后兼容增强，例如播报格式、字段、口径、排序、异常规则、文档和 skill 的增强。
- `v1.x.0`：功能版本，例如新增一类完整播报、接入新数据源、新机器人场景、群聊/单聊模式扩展。
- `v2.0.0`：破坏性升级，例如重做卡片模板、替换发送链路、废弃旧配置或改动核心数据口径。

## v1.0.0 - 2026-06-21

### 新增能力

- 建立钉钉互动卡片发送链路，使用 `StandardCard` 发送机器人单聊报表。
- 支持 Markdown 卡片和图片表格两种播报模式，并通过配置切换。
- 建立退费率播报样板，包括标题、时间、口径说明、表格、异常关注和真实卡片验证流程。
- 图片表格支持固定列、表头配色、斑马纹、总计行、条件色、`clipPath` 防遮挡。

### 数据口径

- 退费率播报默认按当日 `00:00` 至当前整点统计，对比昨日同整点窗口。
- `00:00:00` 至 `00:59:59` 是特殊整点窗口：统计昨日全天，对比前日全天。
- 分时对比是通用规则：对比周期必须取对应的同一分时窗口，例如今日 `00:00-09:00` 对比昨日 `00:00-09:00`。
- DataFinder 非全天窗口使用 `granularity: "hour"` 并汇总已完成小时桶；全天窗口使用 `granularity: "day"`。

### 发送目标和触发方式

- 当前目标为小灰龙-运营助手机器人单聊。
- 本地常驻服务每小时整点自动推送。
- 保留单次脚本入口 `npm run refund-report:once`，用于测试卡片和后续调度迁移。

### 配置项变化

- `REFUND_REPORT_RENDER_MODE=markdown|image` 控制 Markdown 或图片表格模式。
- `REFUND_REPORT_CARD_TEMPLATE_ID=StandardCard` 使用标准互动卡片模板。
- `REFUND_REPORT_TIMEZONE=Asia/Shanghai` 控制自然日和整点窗口。
- `REFUND_REPORT_THRESHOLD_PERCENT=10` 控制退费率环比关注阈值。

- 已验证互动卡片正文完整渲染，图片表格可见。

## v1.0.1 - 2026-06-21

### 新增能力

- 图片表格指标单元格从两行升级为三行：当前值、昨日同分时对比、前日/昨日全日参考值。
- 新增 `c0收入` 列，位置在 `支付数` 后、`退费金额` 前。
- `c0收入` 按 `order_suc_back` 事件、`cycle = 0` 过滤、`amount * events` 汇总，覆盖当前分时、昨日同分时、前日全天和昨日全天四套窗口。
- 图片表格列顺序更新为 `企业名称 | 退费率 | 退费后金额 | 支付金额 | 支付数 | c0收入 | 退费金额 | 退费数`。

### 数据口径

- 继续沿用 v1.0.0 的分时对比规则：非 00 点窗口必须用当前周期 `00:00-当前整点` 对比基准周期 `00:00-同整点`，不得取基准周期全天。
- 继续沿用 00 点特殊窗口：`00:00:00` 至 `00:59:59` 使用昨日全天对比前日全天。
- 新增全日参考窗口：第三行只展示前日全天与昨日全天值，不计算环比。
- DataFinder 非全天窗口继续使用 `granularity: "hour"` 并汇总小时桶；全日参考窗口使用 `granularity: "day"`。

### 发送目标和触发方式

- 推送目标、触发频率、互动卡片发送链路不变。
- 本地常驻服务仍按每小时整点自动推送；单次入口 `npm run refund-report:once` 仍可用于测试卡片。
- Markdown/image 模式开关继续保留，本次只增强图片表格模式。

### 配置项变化

- 未新增环境变量。
- `REFUND_REPORT_RENDER_MODE=image` 时启用三行图片表格；`markdown` 模式保持 v1.0.0 文本格式。

- 已更新图片表格测试，验证 `c0收入` 列顺序、三行单元格、全日参考值、`clipPath` 和 PNG 生成。
- 已运行 `D:\nodejs24\npm.cmd test`，通过 `83 pass / 0 fail`。
